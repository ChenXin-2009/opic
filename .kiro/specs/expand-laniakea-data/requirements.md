# 扩展拉尼亚凯亚超星系团数据 - 需求文档

## 1. 项目概述

### 1.1 背景
当前项目的拉尼亚凯亚超星系团(橙色层级)可视化使用Cosmicflows-3数据集的部分数据,导致显示的超星系团分布不完整。用户观察到GaiaSky显示了更完整的超星系团分布("两边"),而当前实现只显示了"一边"。

### 1.2 目标
扩展Cosmicflows-3数据集,获取完整的17,669个星系距离数据,以实现更完整的超星系团可视化,匹配GaiaSky的显示效果。

### 1.3 数据来源
- **主要数据源**: VizieR J/AJ/152/50 (Cosmicflows-3完整目录)
- **数据规模**: 17,669个星系距离测量
- **坐标系统**: 超星系坐标系(Supergalactic Coordinates)
- **数据格式**: 与现有laniakea_cosmicflows3.txt保持一致

## 2. 用户故事

### 2.1 作为天文爱好者
我希望看到完整的拉尼亚凯亚超星系团及其周边超星系团的分布,这样我可以更好地理解宇宙的大尺度结构。

**验收标准**:
- 可视化显示的超星系团数量显著增加
- 超星系团分布在各个方向上更加均匀
- 与GaiaSky的显示效果相当

### 2.2 作为开发者
我希望数据处理流程保持一致性,这样新数据可以无缝集成到现有系统中。

**验收标准**:
- 新数据使用与现有数据相同的二进制格式
- 数据加载器无需修改即可处理新数据
- 渲染器可以正确显示扩展后的数据

### 2.3 作为性能优化者
我希望扩展数据后系统性能不会显著下降,保持流畅的用户体验。

**验收标准**:
- 帧率保持在30 FPS以上
- 内存使用不超过2GB
- LOD系统正确工作,远距离时减少渲染负担

## 3. 功能需求

### 3.1 数据获取
- **FR-1.1**: 从VizieR下载Cosmicflows-3完整数据集(J/AJ/152/50/table3)
- **FR-1.2**: 数据应包含以下字段:
  - PGC编号(星系目录编号)
  - 距离(Mpc)
  - 坐标(SGLON, SGLAT - 超星系坐标)
  - 速度(HV, Vgsr, Vls, Vcmb)
  - 星系团归属(Abell, GName)
  - 亮度(Bmag, Ksmag)

### 3.2 数据处理
- **FR-2.1**: 将下载的数据转换为超星系笛卡尔坐标(X, Y, Z)
- **FR-2.2**: 识别并聚类超星系团结构
  - 使用空间聚类算法(如DBSCAN或Friends-of-Friends)
  - 计算每个超星系团的中心位置、半径、成员数量
- **FR-2.3**: 生成二进制数据文件,格式与现有laniakea.bin一致:
  ```
  - 名称表大小 (uint16)
  - 名称表 (每个名称: uint8长度 + 字符串)
  - 超星系团数量 (uint16)
  - 超星系团数据 (每个超星系团):
    - centerX, centerY, centerZ (float32)
    - radius (float32)
    - memberCount (uint16)
    - richness (uint8)
    - nameIndex (uint8)
    - hasVelocity (uint8)
    - velocityX, velocityY, velocityZ (float32, 可选)
    - 成员星系 (每个: x, y, z float32)
  ```

### 3.3 数据验证
- **FR-3.1**: 验证数据完整性
  - 检查星系数量是否接近17,669
  - 验证坐标范围合理性
  - 确认超星系团分布的对称性
- **FR-3.2**: 与现有数据对比
  - 确认已有超星系团仍然存在
  - 验证新增超星系团的合理性

### 3.4 可视化增强
- **FR-4.1**: 渲染器应能处理增加的超星系团数量
- **FR-4.2**: LOD系统应根据超星系团数量自动调整
- **FR-4.3**: 标签系统应优先显示重要的超星系团(如Laniakea, Virgo, Coma, Perseus等)

- **NFR-3.2**: 数据来源应在metadata.json中记录

### 4.4 数据质量
- **NFR-4.1**: 数据精度保持与原始Cosmicflows-3一致
- **NFR-4.2**: 坐标转换误差小于1%

## 7. 风险与缓解

### 7.1 数据获取风险
**风险**: VizieR服务可能限制大量数据下载
**缓解**: 使用VizieR的批量下载API,或分批下载

### 7.2 性能风险
**风险**: 数据量增加可能导致性能下降
**缓解**: 优化LOD系统,实现更激进的剔除策略

### 7.3 数据质量风险
**风险**: 自动聚类可能产生不准确的超星系团
**缓解**: 与已知超星系团目录对比验证,手动调整参数
